CHECKLIST DE CUMPLIMIENTO - TABLEFLOW

1. RESUMEN DEL PROYECTO
Este proyecto es una aplicación web de gestión para restaurante.
Incluye:
- frontend en Flask
- backend en FastAPI
- microservicio gRPC para cocina
- base de datos MySQL
- ejecución con Docker Compose

Objetivo funcional:
- registrar e iniciar sesión de usuarios
- gestionar menú
- crear órdenes
- actualizar estados de órdenes
- enviar notificaciones en tiempo real por WebSocket

6. CÓMO PROBABLEMENTE VAN A EVALUAR
Lo más probable es que evalúen por funcionamiento y por estructura.
Puntos esperables de revisión:
- que docker compose up --build levante todo
- que el frontend cargue
- que el backend responda en /docs y /health
- que se pueda iniciar sesión
- que se pueda ver el menú
- que se puedan crear órdenes
- que se puedan cambiar estados válidos
- que las capas estén separadas como lo pide el TXT
- que el backend sea quien hable con kitchen por gRPC
